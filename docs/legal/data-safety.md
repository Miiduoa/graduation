## Campus Helper (校園助手)

**Version:** 1.0
**Last Updated:** 2026-03-20

## Overview

This document maps Campus Helper's data collection practices to Google Play's Data Safety requirements. All information below is accurate as of the app's current version and reflects actual data handling practices.

---

## 1. Data Collection Summary

### Data Collected: YES

Campus Helper collects certain personal and device data to provide its services. Users can control most data collection through privacy settings.

### Data Deletion: YES

Users can request deletion of their account and associated data through Settings > Privacy > Delete Account. Most data is automatically deleted after specified retention periods.

## 4. Data Security

### 4.1 Encryption Standards

**In Transit:**
- TLS 1.3 for all network communications
- Encrypted push notifications
- Secure WebSocket (WSS) for real-time features

**At Rest:**
- AES-256 encryption for sensitive data (passwords, tokens)
- Database-level encryption for user data
- File-level encryption for backups

**Password Storage:**
- bcrypt hashing with salt
- Plain-text passwords NEVER stored
- Passwords NEVER logged or backed up

### 4.2 Authentication Security

- Secure password requirements enforced
- Optional two-factor authentication (TOTP)
- Session tokens with limited lifetime
- Automatic logout after inactivity (15 minutes default)
- Login attempt rate limiting (5 attempts, 15-minute lockout)

### 4.3 Infrastructure Security

- Firebase security with Google's infrastructure
- DDoS protection via Google Cloud
- Web Application Firewall (WAF)
- Automated backups with encryption
- Regular security patches and updates

### 4.4 Vulnerability Management

- Regular security audits
- Penetration testing (annual)
- Responsible disclosure program
- Contact: security@campushelper.app
- Bug bounty program (information: https://campushelper.app/security)

## 5. User Controls and Privacy Settings

### 5.1 Location Controls

**Settings Menu:** Settings > Privacy > Location Services

- **Toggle:** On/Off (default: OFF)
- **Background Location:** On/Off (requires foreground permission first)
- **Location History Retention:** 7 days, 30 days, or Manual delete
- **Location Share with Others:** On/Off (show "on campus" status)

**Effect of Disabling:**
- Map shows user's location but doesn't store history
- Location-based notifications disabled
- Campus service locations still show (static, not user-relative)
- Classroom finder still works (but shows walk-in mode)

### 5.2 Camera and Photo Permissions

**Settings Menu:** System Settings > Apps > Campus Helper > Permissions

- **Camera:** On/Off
- **Photo Library:** Full Access, Selected Photos, or None
- **Microphone:** On/Off (if applicable)

**Effect of Disabling:**
- QR code scanning unavailable (manual input alternative provided)
- Profile photo must be selected or skipped
- Photo uploads disabled (text-only posts still available)

### 5.3 Notification Controls

**Settings Menu:** Settings > Notifications

- **Course Reminders:** On/Off
- **Campus News:** On/Off
- **Social Updates:** On/Off
- **Quiet Hours:** Configurable (default 21:00-08:00)
- **Notification Limits:** 5, 10, 20, or Unlimited per day
- **Sound:** On/Off per category

**Mandatory Notifications (cannot disable):**
- Security alerts
- Payment confirmations
- Critical campus alerts

### 5.4 Analytics and Tracking

**Settings Menu:** Settings > Privacy > Analytics

- **App Analytics:** On/Off
- **Crash Reporting:** On/Off
- **Feature Usage Tracking:** On/Off

**Effect of Disabling:**
- No usage data sent to Firebase
- Crash reports not sent
- App still fully functional
- App improvements slower

### 5.5 Social and Privacy Controls

**Settings Menu:** Settings > Privacy > Social

- **Profile Visibility:** School-wide, Study Groups Only, or Private
- **Show Online Status:** On/Off
- **Allow Messages from Unknown Users:** On/Off
- **Location Sharing with Friends:** On/Off

### 5.6 Data Requests

**Access and Export:**
- Settings > Privacy > Export My Data
- Generates ZIP with all personal data
- Format: JSON or CSV
- Available within 5 minutes of request

**Account Deletion:**
- Settings > Privacy > Delete Account
- 30-day cooling-off period
- Can cancel within 30 days
- Permanent deletion after 30 days

**Support Data Requests:**
- Can request specific data types
